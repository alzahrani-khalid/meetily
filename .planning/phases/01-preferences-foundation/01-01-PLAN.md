<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Tauri IPC (frontend → Rust) | Untrusted input arrives as `UserPreferencesPatch` JSON from React code running in the Tauri webview |
| Rust → SQLite | Every write goes through `sqlx` with bind parameters — no raw SQL string interpolation |
| SQLite file on disk | Local single-user desktop app; filesystem trust boundary is out of scope (same as rest of app) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-01 | Tampering (Information Disclosure) | SQLite file at rest — attacker with filesystem access writes directly to `user_preferences`, bypassing Rust-level invariant | **accept** | Out of scope — local-only threat model, single-user desktop app, no secrets in this row (only locale strings). Documented as known gap per CONCERNS.md #4. **Severity: LOW** — does not block. Bound to T2 verification only because filesystem-level tampering is out of scope for automated test. |
| T-1-02 | Repudiation (Race / Desync) | `apply_patch_atomic` cache-then-commit ordering inversion — concurrent reader could see pre-commit cache state | **mitigate** | Enforced in-phase: `apply_patch_atomic` commits FIRST, `set_user_preferences` updates `PREFS_CACHE.write().await` AFTER `tx.commit()` returns Ok. Violation of this ordering causes T3 (rollback invariance) to fail because on tx failure the cache must be untouched. **Severity: MEDIUM** — mitigated. Bound to T3. |
| T-1-03 | Tampering (Input Validation Bypass) | Reject branch bypassed by malformed Tauri IPC input — frontend sends `{provider: 'parakeet'}` while Arabic is active and expects the cache to mutate | **mitigate** | Enforced in-phase: `apply_patch_atomic` runs `invariant::check_reject_branch(&patch, &merged)` BEFORE `pool.begin().await` is called. Serde deserialization catches type-mismatch errors. T4 asserts `matches!(result, Err(PreferencesError::InvalidCombination { .. }))` via exact variant match (not generic `is_err()`). **Severity: MEDIUM** — mitigated. Bound to T4. |
| T-1-04 | Denial of Service (Deadlock) | Two concurrent `set_user_preferences` calls deadlock on `PREFS_CACHE.write().await` | **mitigate** | Enforced in-phase: `read()` returns an owned `UserPreferences` by clone (not a guard), so no guard is held across `.await`. Write path acquires the lock only once per call after the transaction commits. T5 exercises `tokio::try_join!(set1, set2)` with a 2-second timeout. **Severity: MEDIUM** — mitigated. Bound to T5. |

All threats map 1:1 to VALIDATION.md Per-Task Verification Map Threat Ref column.
